# Feature Intake

Every implementation request enters intake before code.

## Input Types

| Type | Use When |
| --- | --- |
| New spec | New product idea or complete spec. |
| Spec slice | Selected behavior from accepted spec. |
| Change request | Refinement, bug, behavior change. |
| New initiative | Multi-story product area. |
| Maintenance | Dependency, security, performance, operations. |
| Harness improvement | Process, template, integration, agent instruction. |

## Risk Lanes

- tiny: docs, copy, narrow non-contract work.
- normal: story-sized behavior with bounded blast radius.
- high-risk: auth, authorization, data model, migration, data loss, payment, provider, security, privacy, public API, weak proof, or multi-domain change.
